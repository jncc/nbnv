code description

POST

Post a response to an existing organisation join request, can be either an accept / deny (org admin) or a withdraw (requesting User), if the call has is incorrect (json passed is wrong) return a BAD_REQUEST response. If the request is made by someone who is neither the requesting user or an org admin of the organisation in this request then a FORBIDDEN response should be shown.

name description type default
user The current user (Injected Token no need to pass) tokenorganisationjoinrequestuser

Response Body

element: (custom)

application/xml

code description

202 Accepted or denied join request successfully, but could not send notification email
